Quick Glossary — what do these words mean?

🧑‍💼
Original Realtor
The real, main record for a realtor that we trust and show to users. Think of it like the "official profile" — there is only one original per real person.
👯
Duplicate
A copy of the same realtor that came from a different data source or was imported twice. It has the same person behind it but we hide it so users only see the original.
🗂️
Duplicate Group
A family of records that all represent the same real-world person — one original plus all its duplicates. Like a folder that groups all the copies together.
🔑
duplicate_uid
A shared ID tag (like a group badge) placed on every record in the same duplicate group — both the original and all its duplicates carry this tag so the system knows they belong together. Records without this tag have never been linked to any group.

Block 7 — Name-Dedup 95% (Pair Analysis)

What is this? Imagine going through a huge phone book and spotting two entries that look almost identical — same name, same state — and asking: "Are these really the same person listed twice?" That is exactly what this analysis does. It finds pairs of realtor profiles whose names are at least 95% similar and who work in the same state, then checks whether our system has already figured out they are the same person or not.

Every pair falls into one of four buckets:

  • Same Group — the system already knows these two profiles belong to the same person. Think of it as two entries already stapled together in our filing cabinet. Nothing needs to be done. ✅
  • Cross-Group — each profile is already in its own separate folder, but the two folders may actually belong to the same person. These are candidates for merging two existing folders into one.
  • Both Without Group — neither profile has ever been put into any folder. The system has never spotted this potential match before. These are brand-new discoveries that need attention.
  • Mixed — one profile is already in a folder, the other is a lone entry sitting outside. The lone entry may simply need to be moved into the existing folder.

The bottom three buckets together make up the NEW Merge Pairs — matches the system has not yet acted on. The percentage shown is calculated against the total number of realtor profiles in the database (not just the pairs found here), so it tells you: "Out of all realtors we have, what share could potentially be affected by unresolved duplicates?"
